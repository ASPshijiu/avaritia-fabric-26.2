# 1.0.0-rc.1 验证报告

验证日期：2026-07-21  
验证平台：macOS 26.5.2（Apple M5）、OpenJDK 25.0.3  
目标：Minecraft Java 26.2、Fabric Loader 0.19.3、Fabric API 0.155.2+26.2

## 自动化门禁

`JAVA_HOME=/opt/homebrew/opt/openjdk@25 ./gradlew clean build` 通过，包含：

- 88/88 GameTest 通过。
- 568 个 JSON 全部可解析。
- 96 个生产配方与 96 个配方解锁进度一一对应。
- 27 个玩家可获得方块与 27 个方块战利品表一一对应。
- `en_us` 与 `zh_cn` 的 208 个键完全一致。
- 模组命名空间内的模型与纹理引用全部可解析。
- 50 台中子收集器与 50 台满料压缩机同时运行时，逻辑耗时 p95 为 `0.692458 ms/tick`，低于 `5 ms/tick` 门槛。

逐个物品、方块和奇点的合成/产出、使用、功能、客户端与专服结论记录在 [FEATURE_MATRIX.md](FEATURE_MATRIX.md)。自动化不仅检查注册，也对配方形状、错误输入拒绝、容器保存、物品组件、工具/武器/护甲能力、实体、网络同步与数据重载进行行为断言。

## 真实发行 JAR 验证

为排除“开发源集可运行但发行包缺文件”，下列实例的类路径明确排除了项目 `build/classes` 与 `build/resources`，只加载构建出的模组 JAR 和 Fabric API。

### 专用服务器

- 全新目录启动成功并创建世界，随后对同一世界重启成功。
- 两次都加载 42 个模组、1681 个总配方、1784 个总进度和 14 个生产奇点，并到达 `Done`。
- 两次均使用 `stop` 正常退出，所有维度区块保存完成。

### 客户端与单人世界

- OpenGL：完成资源重载、声音与全部图集创建，创建并进入单人世界，集成服务器保存成功。
- Vulkan：使用 `Vulkan 1.2.334 / MoltenVK 1.4.2` 和 Apple M5，完成资源重载、全部图集创建、1681 个配方、1784 个进度和 14 个生产奇点加载，并进入已有单人世界。
- 源码未引用 `org.lwjgl.opengl.*`，两条渲染路径均由 Minecraft/Fabric 后端无关接口驱动。

### 多人

- 一台发行 JAR 客户端成功加入本地专用服务器。
- 客户端断线后，以同一 UUID 重新连接成功，服务端恢复到相同位置。
- 两台客户端同时在线未在当前无启动器、无法枚举原始 Java/LWJGL 窗口的自动化环境中完成；因此该项保留为 RC 人工回归项。

## 非模组错误与预期警告

- 离线测试令牌会触发 Mojang/Yggdrasil/Realms 请求超时或鉴权失败；它们发生在外部服务调用，不影响本地世界、资源加载或专服验证。
- `eternal_singularity` 与 `infinity_catalyst` 的配方书自动摆放警告来自动态材料清单，手动合成路径已通过。
- MoltenVK 的 Metal 特性提示属于实验性 Vulkan 驱动信息；本次验证未出现模组资源缺失、粉黑材质或崩溃。

## 发布结论

当前实现满足功能矩阵和 RC 自动化门禁，可发布 `1.0.0-rc.1`。正式版前仍建议由两台可控的真实客户端补做同时在线人工回归，并扩大不同 GPU/驱动的 Vulkan 覆盖。
